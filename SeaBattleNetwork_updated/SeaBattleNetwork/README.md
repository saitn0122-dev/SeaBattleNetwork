# Морской бой — WinForms, сеть, MySQL, боты

Готовый учебный проект по ПМ2 «Прикладное программирование».

## Что входит

- Игровое поле 10×10.
- Автоматическая расстановка кораблей.
- Ходы игрока по клеткам PictureBox.
- Элементы Button, PictureBox, Label, MenuStrip, TabControl, DataGridView.
- Timer на 30 секунд для контроля времени хода.
- Режим игры с ботом.
- 3 сложности бота:
  - Easy — случайные выстрелы.
  - Medium — после попадания стреляет рядом.
  - Hard — использует шахматный поиск и добивание.
- Сетевой обмен через HttpClient и отдельный локальный HTTP-сервер.
- MySQL для сохранения статистики.
- SQL-скрипт базы данных.
- ER-диаграмма в docs/ER-diagram.md.

## Как запустить

1. Откройте `SeaBattleNetwork.sln` в Visual Studio 2022.
2. Установите .NET 8 SDK.
3. Запустите MySQL и выполните скрипт:
   `database/seabattle_db.sql`
4. В файле `SeaBattle.WinForms/StatisticsRepository.cs` при необходимости измените строку подключения:
   ```csharp
   Server=localhost;Port=3306;Database=seabattle_db;Uid=root;Pwd=12345;
   ```
5. Для сетевой игры сначала запустите проект `SeaBattle.Server`.
6. Затем запустите два клиента `SeaBattle.WinForms`.
7. На первом клиенте нажмите «Создать сетевую».
8. На втором клиенте введите Game ID и нажмите «Подключиться».

## Пояснение для защиты

### GameEngine.cs
Содержит основную игровую логику: поле 10×10, расстановку кораблей, проверку попаданий, промахов, уничтожения корабля и победы. Корабли хранятся как список координат.

### MainForm.cs
Отвечает за интерфейс Windows Forms. Игровое поле создается динамически из массива `PictureBox`. Цвет клетки меняется в зависимости от состояния: вода, корабль, промах, попадание, уничтоженный корабль. Также здесь используется `Timer` для ограничения времени хода.

### BotPlayer.cs
Реализует бота с тремя сложностями. Простой бот стреляет случайно, средний пытается добить корабль после попадания, сложный использует шахматный алгоритм поиска.

### NetworkClient.cs
Содержит работу с `HttpClient`: создание игры, подключение, отправка хода и получение последнего хода. Все методы асинхронные, поэтому интерфейс не зависает.

### StatisticsRepository.cs
Работает с MySQL через пакет `MySqlConnector`. После окончания игры сохраняет результат и загружает таблицу статистики в `DataGridView`.

### SeaBattle.Server
Минимальный HTTP-сервер на ASP.NET Core. Он хранит созданные игры и ходы игроков. Это демонстрирует сетевой режим и обмен данными между двумя клиентами.

## Обновление: синхронизация сетевой игры и ручная расстановка

В этой версии добавлено:

- ручная расстановка кораблей на своём поле;
- кнопка «Повернуть» для смены направления корабля;
- синхронизация ходов двух игроков через сервер;
- поддержка игры по локальной сети.

### Как вручную расставить корабли

1. Нажмите кнопку «Ручная расстановка».
2. Нажимайте по клеткам на поле «Ваше поле».
3. Корабли ставятся по порядку: 4, 3, 3, 2, 2, 2, 1, 1, 1, 1.
4. Кнопка «Повернуть» меняет направление корабля: горизонтально / вертикально.
5. Если корабль нельзя поставить, программа покажет сообщение внизу.
6. После установки всех кораблей можно начинать игру.

### Как играть по локальной сети

1. На компьютере, который будет сервером, запустите проект `SeaBattle.Server`.
2. Сервер слушает адрес `http://0.0.0.0:5100`, поэтому он доступен другим устройствам в одной Wi-Fi/локальной сети.
3. Узнайте IP компьютера-сервера через команду:

```cmd
ipconfig
```

Например IP: `192.168.0.15`.

4. На втором компьютере перед запуском клиента задайте адрес сервера:

```cmd
set SEABATTLE_SERVER_URL=http://192.168.0.15:5100
SeaBattle.WinForms.exe
```

На компьютере-сервере можно оставить `localhost`.

5. Первый игрок нажимает «Создать сетевую» и сообщает ID игры второму игроку.
6. Второй игрок вводит ID и нажимает «Подключиться».
7. Ходы синхронизируются автоматически: один клиент отправляет выстрел, второй получает его, обрабатывает попадание/промах и отправляет результат обратно.

### Что говорить на защите

Сетевая игра работает по клиент-серверной схеме. Клиенты WinForms используют `HttpClient`, а сервер хранит сообщения между игроками. Когда первый игрок стреляет, на сервер отправляется сообщение `Shot`. Второй клиент получает это сообщение через таймер опроса, применяет выстрел к своему полю и отправляет обратно результат: `Miss`, `Hit`, `Killed` или `Win`. Благодаря этому оба клиента синхронизируют ход игры.
